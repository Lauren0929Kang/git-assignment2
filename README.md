# 기관차 문제 (The Locomotive Problem)

목격한 기관차 번호 하나로 그 회사가 가진 전체 기관차 수를 베이즈 추론으로 추정해보는 코드.

## 문제

철도 회사 기관차에 1번부터 N번까지 번호가 붙어 있다고 하자.
길에서 60번 기관차를 봤다면, 이 회사는 기관차를 총 몇 대나 가지고 있을까?

관측값 하나(60번)만으로 전체 규모를 추정하는 문제다. "독일 전차 문제"라는 이름으로도 알려져 있다.

## 원리

베이즈 정리로 관측값을 반영해서 각 가설(전체 기관차 수)의 확률을 갱신한다.

- `init` : 모든 후보에 같은 확률 부여 (균등 사전분포)
- `likelihood` : 전체가 N대일 때 60번을 볼 확률 = 1/N
- `update` : 관측값 반영해서 확률 갱신
- `normalize` : 확률 합이 1이 되도록 조정
- `posterior_mean` : 사후분포 평균 계산

전체 수가 목격한 번호보다 작으면 애초에 불가능하니까 likelihood가 0이 된다.


## 결과

사후확률분포 그래프는 posterior.png 로 저장된다.

60대 미만은 확률이 0이고(60번을 봤으니 그보다 적을 수 없음), 60에서 제일 높았다가 뒤로 갈수록 천천히 줄어든다.

여기서 중요한 부분은, 상한값을 얼마로 잡느냐에 따라 평균 추정치가 207 → 333 → 552로 꽤 크게 움직인다는 점이다. 데이터는 60번 하나뿐인데 결과가 이만큼 달라지는 건 사전분포를 어떻게 두느냐가 그대로 반영되기 때문. 데이터가 적을 때 사전분포가 얼마나 중요한지 보여주는 예시다.
